        Knowledge base

        Common questions

        The DOM (Document Object Model) is the browser's live map of your HTML. JavaScript can reach into it, grab elements, change their text, add/remove classes, create new nodes, or delete existing ones — all without reloading the page. It's the core superpower of frontend development.

        textContent sets plain text and is safe from XSS attacks — it treats everything as a literal string. innerHTML parses raw HTML, which is powerful but dangerous if you're injecting user data. Always default to textContent for data you don't control.

        Use const by default — it signals that this binding won't be reassigned, which makes your code easier to reason about. Switch to let only when you know the value will be updated (counters, loop variables, toggles). Avoid var entirely; it's function-scoped and prone to bugs from hoisting.

        Separating concerns keeps your code clean. JavaScript handles behavior (toggling a class), while CSS handles visuals (what that class looks like, including transitions). When you inline styles from JS, you lose the ability to animate with CSS transitions and end up with logic spread across two places.

        Instead of attaching a listener to every child element, you attach one listener to a parent and use event.target to figure out which child was clicked. This is far more efficient — it works for elements added dynamically, and reduces memory usage. Essential for things like todo lists or comment threads.
